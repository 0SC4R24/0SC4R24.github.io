◆ blockResult

enum blockResult

Representa los posibles resultados de un intento de bloqueo.

Valores posibles:

  • fail: El bloqueo falla y el atacante impacta normalmente.
  • success: Bloqueo exitoso que reduce o evita el daño.
  • critical: Bloqueo crítico que puede devolver una ventaja (por ejemplo, contraataque).

Ejemplo:

blockResult r = player.block();
if (r == critical) { // iniciar contraataque }

◆ defendResult

enum defendResult

Representa los posibles resultados de una acción de defensa.

Valores:

  • failed: No se defendió, el daño se aplica normalmente.
  • dodged: Evitó el ataque por completo.
  • blocked: Bloqueó parcialmente el ataque.
  • parried: Parry exitoso (posible efecto adicional en el atacante).

◆ playerClass

enum playerClass

Representa las clases disponibles para un jugador.

Las clases afectan comportamiento en ataque/defensa y pueden cambiar la lógica en clases derivadas (Warrior, Archer, Paladin, Mage).
