Detailed Description

Definición de la clase Mage, especialista en daño mágico y curación.

La clase Mage hereda de Player y provee comportamiento específico para lanzadores de hechizos: ataques basados en intelligence y una habilidad de curación interna.

El mago puede "desperdiciar" un turno para curarse en lugar de atacar. Esto significa que, en el flujo de juego, una llamada a attack() podría optar por realizar la acción de curación (consumir el turno) en lugar de infligir daño al objetivo.

Ejemplo de uso:

Mage m("Gandalf");
// En cada turno, la IA/decisión del mago puede elegir entre atacar o curarse
int dmg = m.attack();
if (dmg == 0) {
// Interpretación común: el mago se curó ese turno en lugar de atacar
}
std::cout << "Daño mágico (o 0 si se curó): " << dmg << std::endl;

◆ attack()

int Mage::attack ( )
overridevirtual

Implementación de ataque para magos.

Este método sobrescribe Player::attack() y debe calcular daño mágico habitualmente basado en la estadística intelligence.

El mago puede elegir gastar un turno para curarse en lugar de atacar. En esa situación la implementación típica es:

  • llamar a heal() para restaurar HP;
  • devolver 0 como daño infligido en ese turno (señalando que el turno se usó para curación en lugar de ofensiva).
Returns
int Daño mágico calculado por el ataque, o 0 si el mago decidió curarse ese turno.

Ejemplo:

Mage m("Orion");
int dmg = m.attack();
if (dmg == 0) {
std::cout << "Orion se curó este turno." << std::endl;
} else {
// Aplicar daño: target.takeDamage(dmg, &m);
}
