Detailed Description

Definición de la clase Paladin, un jugador tanque con capacidades de apoyo sagrado.

Según la descripción del proyecto (README), el Paladin obtiene un beneficio en su siguiente ataque si su defensa (bloqueo o parry) es exitosa. Esta clase hereda de Player y adapta la lógica de defensa y ataque para reflejar ese comportamiento.

Ejemplo de uso:

Paladin p("Uther");
// Durante la defensa, si consigue blocked o parried, su siguiente ataque
// tendrá un bono sagrado.
defendResult res = p.defend();
if (res == blocked || res == parried) {
// El siguiente ataque de p estará fortalecido automáticamente
}
int dmg = p.attack(); // daño ya con posible holyBuff aplicado

int Paladin::attack ( )
overridevirtual

Implementación de ataque para el Paladin.

Este método sobrescribe Player::attack() y debe aplicar el holyBuff (si existe) al daño calculado y luego resetear holyBuff a 0. El bono representa el efecto sagrado acumulado tras una defensa exitosa.

Returns
int Daño calculado (incluyendo holyBuff si procede).

Ejemplo:

int d = pal.attack();
std::cout << "Daño paladín: " << d << std::endl;

◆ defend()

defendResult Paladin::defend ( )
overridevirtual

Lógica de defensa específica del Paladin.

Si la defensa resulta en blocked o parried, este método debe establecer holyBuff para que el siguiente ataque cause daño adicional. Devuelve el resultado de la defensa (failed, dodged, blocked, parried), igual que en Player.

Returns
defendResult Resultado de la defensa.

Ejemplo:

defendResult r = pal.defend();
if (r == parried) {
// holyBuff aplicado internamente
}

Reimplemented from Player.
